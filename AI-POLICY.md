# AI Policy

Open Source Europe (OSE) is a pro-human community. AI tools are instruments; humans are the only authors of contributions. Agents are assistants — they are never co-authors.

## Scope

This policy applies to all contributors across all OSE repositories — code, documentation, governance proposals, issues, and pull requests.

## Disclosure Requirements

### Pull Requests

If you used generative AI tooling to help create a pull request, you must disclose it in the PR description using the format from the PR template:

```
Generated-by: <Agent Name and Version> following [AI Policy](https://github.com/opensourceeurope/.github/blob/main/AI-POLICY.md)
```

### Issues

Issues created with AI-assisted content must note it in the dedicated field of the issue template.

## Using Generative AI Tools

Using AI tools to help create contributions is permitted as long as you follow these guidelines:

1. **Review everything.** You must review and understand all content generated by AI tools before including it in your contribution. Do not blindly trust or copy-paste AI output.

2. **Verify quality.** Ensure generated content adheres to the project's standards and passes all relevant checks. Do not rely on maintainers to catch errors introduced by AI tools.

3. **Remove unrelated changes.** AI tools often introduce changes unrelated to the problem you are solving. Remove them before submitting. Relying on maintainers to spot unrelated changes adds unfair burden on them.

4. **Be ready to explain.** Be prepared to explain and justify any AI-generated content if asked by maintainers or reviewers.

5. **You are responsible.** The final responsibility for everything in your contribution lies with you, regardless of how it was produced.

6. **Verify license compatibility.** Ensure no AI-generated content introduces material under an incompatible license.

Blindly copy-pasting AI output introduces security and stability risks. Maintainers who identify such behaviour will close the related pull request without review and note the reason in a comment. Repeated violations may result in the contributor being blocked from making further contributions.

## Prohibited Uses

The following are human-only responsibilities and must never be delegated to or performed by AI tools:

- Casting governance votes or approvals
- Making Code of Conduct enforcement decisions
- Security vulnerability assessments and sign-offs
- Maintainer nominations and access grants

## Enforcement

Violations of this policy are treated as Code of Conduct violations and handled under the same process defined in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). To report a violation, email [lukasz@opensourceeurope.org](mailto:lukasz@opensourceeurope.org).
